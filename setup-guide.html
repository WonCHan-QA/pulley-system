1
GitHub 저장소 생성 및 파일 업로드
GitHub Pages가 웹 호스팅 역할을 합니다. 무료이며 HTTPS가 자동 적용됩니다.
  1. github.com 접속 → 로그인 (계정 없으면 무료 가입)
  2. 우측 상단 ＋ → New repository 클릭
  3. Repository name: pulley-system (또는 원하는 이름)
  4. Public 선택 → Create repository
  5. 다운받은 zip 압축 해제 후, 폴더 안의 파일 3개를
    GitHub 페이지의 uploading an existing file 링크로 업로드:
    • index.html — 메인 관리 시스템
    • drawing.html — QR 스캔 도착 페이지
    • docs/setup-guide.html — 이 가이드
  6. Commit changes 클릭
  7. 저장소 Settings → Pages → Branch: main → Save
  8. 약 1분 후 https://[사용자명].github.io/pulley-system/ 접속 확인
💡
GitHub 사용이 어렵다면, Netlify Drop (netlify.com/drop)에 파일을 드래그하면 즉시 URL이 생성됩니다.
2
Google Sheets 생성
데이터가 저장되는 스프레드시트를 만듭니다.
  1. sheets.new 접속하여 새 스프레드시트 생성
  2. 파일명을 한국정공_풀리도면관리로 변경
  3. 나중에 Apps Script가 시트를 자동으로 생성해 줍니다
💡
시트 이름/구조는 Apps Script가 자동 생성하므로 따로 설정 불필요
3
Google Apps Script 배포
Apps Script가 웹 API 서버 역할을 합니다. 스프레드시트에서 실행합니다.
  1. 생성한 Google Sheets에서 확장 프로그램 → Apps Script 클릭
  2. 기존 코드(function myFunction...) 전체 삭제
  3. 다운받은 zip에 포함된 Code.gs 내용을 붙여넣기
  4. 상단 💾 저장 아이콘 클릭
  5. 배포 → 새 배포 클릭
  6. ⚙ 톱니바퀴 → 웹 앱 선택
  7. 설명: 풀리도면관리 API v1
  8. 다음 사용자로 실행: 나
  9. 액세스 권한: 모든 사용자
  10. 배포 클릭 → Google 계정 권한 허용
  11. 배포 URL 복사 (https://script.google.com/macros/s/.../exec)
배포 URL 예시: https://script.google.com/macros/s/AKfycbxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx/exec
⚠
코드를 수정할 때마다 새 배포가 아닌 기존 배포 관리 → 편집으로 버전을 업데이트해야 반영됩니다.
4
웹 앱에서 연결 설정
GitHub Pages에 배포된 웹 앱에서 Apps Script URL을 입력합니다.
  1. https://[사용자명].github.io/pulley-system/ 접속
  2. 설정 화면에서 Apps Script 배포 URL 입력
  3. 사용자 이름, 부서 입력
  4. 연결 및 데이터 가져오기 클릭
  5. 처음 연결 시 Google 시트에 헤더가 자동 생성됩니다
  6. 풀리 목록 → 신규 등록으로 데이터 추가 가능
💡
설정 정보는 브라우저 localStorage에 저장되어 다음 접속 시 자동 연결됩니다
5
도면 파일 연결 (Google Drive)
QR 스캔 시 실제 도면 파일을 열 수 있도록 Google Drive 링크를 연결합니다.
  1. Google Drive에 도면 PDF/DWG 파일 업로드
  2. 파일 우클릭 → 공유 → 링크 복사
  3. 웹 앱 풀리 목록에서 해당 품목 → 상세 또는 편집
  4. 도면 URL 필드에 Google Drive 링크 붙여넣기
  5. 이후 QR 스캔 시 해당 도면이 바로 열립니다
💡
Drive 링크를 뷰어 공유로 설정하면 다운로드 없이 브라우저에서 바로 볼 수 있습니다
Q
QR코드 사용 방법
각 풀리의 QR코드를 생성하고 부착하면 스마트폰으로 즉시 도면을 조회할 수 있습니다.
  1. 웹 앱 → QR관리 탭 이동
  2. 원하는 품목 카드 클릭 → QR코드 생성
  3. PNG 다운로드 → 인쇄 → 해당 풀리 또는 작업지시서에 부착
  4. 스마트폰 카메라로 QR 스캔 → 최신 도면 및 개정이력 확인
💡
QR코드 URL은 GitHub Pages URL + drawing.html?pn=PartNo 형식입니다. 도면이 개정되어도 QR코드를 교체할 필요 없이 시스템에서 버전만 업데이트하면 됩니다.
